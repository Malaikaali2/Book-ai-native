Capstone Evaluation Rubric: Autonomous Humanoid System

Overview​

The Capstone Evaluation Rubric provides a comprehensive framework for assessing the implementation and performance of the autonomous humanoid system. This rubric evaluates the integration of all capstone components including voice processing, task planning, navigation, manipulation, perception, and failure handling. The evaluation framework uses multiple assessment dimensions to ensure thorough evaluation of both technical implementation and practical functionality, with clear criteria for different performance levels.

The rubric is designed to assess not only the individual components but also their integration and coordination, reflecting the complex, multi-faceted nature of autonomous humanoid systems. This evaluation framework ensures that students demonstrate mastery of the complete system lifecycle from design to deployment and validation.

Evaluation Framework​

Assessment Dimensions​

The capstone evaluation encompasses five primary dimensions:

  1. Technical Implementation (30%): Quality of code, architecture, and technical solution
  2. System Integration (25%): Coordination and communication between components
  3. Functional Performance (25%): Achievement of specified system capabilities
  4. Robustness and Safety (10%): Error handling and safe operation
  5. Documentation and Presentation (10%): Quality of documentation and communication

Each dimension is evaluated using a five-point scale:

Assessment Methodology​

Formative Assessment​

  • Weekly Reviews: Instructor feedback on development progress and code quality
  • Milestone Checkpoints: Formal reviews at each project milestone with specific criteria
  • Peer Evaluation: Students review each other's implementations and provide feedback
  • Self-Assessment: Student reflection on development process and learning outcomes

Summative Assessment​

  • System Demonstration: Live demonstration of complete system capabilities
  • Code Review: Comprehensive evaluation of code quality, architecture, and implementation
  • Performance Testing: Validation of all performance requirements and metrics
  • Documentation Review: Assessment of documentation completeness and quality
  • Final Presentation: Professional presentation of project outcomes and lessons learned

Evaluation Process​

  1. Automated Testing: System runs through automated validation pipelines
  2. Manual Review: Instructor evaluation of system quality and implementation
  3. Peer Testing: Other students test and evaluate the systems
  4. Final Assessment: Comprehensive evaluation of all project components

Grading Scale and Calculation​

Overall Grade Calculation​

  • Technical Implementation (30%): Code quality, architecture, and technical solution
  • System Integration (25%): Coordination and communication between components
  • Functional Performance (25%): Achievement of specified system capabilities
  • Robustness and Safety (10%): Error handling and safe operation
  • Documentation and Presentation (10%): Quality of documentation and communication

Letter Grade Scale​

  • A (93-100%): Exemplary work exceeding requirements with professional-level implementation
  • A- (90-92%): Excellent work meeting requirements with high-quality implementation
  • B+ (87-89%): Very good work with minor improvements possible
  • B (83-86%): Good work meeting requirements with solid implementation
  • B- (80-82%): Satisfactory work meeting requirements with some issues
  • C+ (77-79%): Adequate work with noticeable issues
  • C (73-76%): Marginal work with significant issues
  • C- (70-72%): Below requirements with major issues
  • D (60-69%): Inadequate work with substantial issues
  • F (Below 60%): Unacceptable work failing to meet basic requirements

Specific Assessment Tasks​

Task-Based Evaluation​

Each of the following tasks is evaluated according to the rubric criteria:

  1. Voice Command Processing Implementation

    • Natural language understanding accuracy
    • Command execution success rate
    • Response time performance

  2. Task Planning and Execution System

    • Planning algorithm effectiveness
    • Multi-step task coordination
    • Dynamic replanning capabilities

  3. Navigation and Obstacle Avoidance

    • Path planning optimality
    • Obstacle detection and avoidance
    • Navigation accuracy and safety

  4. Object Detection and Manipulation

    • Object recognition accuracy
    • Grasp planning effectiveness
    • Manipulation success rate

  5. Failure Handling and Status Reporting

    • Error detection and classification
    • Recovery strategy effectiveness
    • System status communication

  6. End-to-End Integration

    • Component coordination
    • System-level functionality
    • Performance optimization

Performance Benchmarks​

Minimum Requirements​

  • System Uptime: >95% operational time during demonstration
  • Task Success Rate: >85% for simple tasks, >70% for complex tasks
  • Response Time: <3 seconds for voice command processing
  • Navigation Accuracy: <10cm positioning error
  • Manipulation Success: >80% for standard objects

Excellence Benchmarks​

  • System Uptime: >99% operational time
  • Task Success Rate: >95% for complex tasks
  • Response Time: <1 second for simple commands
  • Navigation Accuracy: <2cm positioning error
  • Manipulation Success: >90% success rate

Assessment Validation​

  • Rubric Review: Regular review and validation of assessment criteria

Academic Integrity and Collaboration Policy​

Individual Work​

  • All system implementation must be original student work
  • Proper attribution required for any external code or resources used
  • Plagiarism will result in immediate failure of the assignment
  • Code sharing between students is prohibited

Acceptable Collaboration​

  • High-level design discussions about system architecture
  • General problem-solving strategies for common robotics challenges
  • Public documentation and robotics resources are allowed
  • Community forums like ROS Answers are acceptable for learning

Required Attribution​

  • All external code sources must be clearly documented
  • Any borrowed algorithms or approaches must be attributed
  • Third-party libraries must be properly credited
  • Collaboration with others must be disclosed

Accommodation and Support​

Technical Support​

  • Office Hours: Regular instructor availability for technical questions

  • Online Resources: Curated list of helpful documentation and tutorials
